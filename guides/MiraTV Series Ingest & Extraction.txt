📺 MiraTV Series Ingest & Extraction
Failure, Recovery, and the New Canonical Path

Status: ✅ Stabilized
Scope: Series ingest → routing → extraction → final artifacts
Outcome: New, resilient extraction path forged out of failure

1️⃣ Executive Summary

The original MiraTV series ingest pipeline failed not due to bugs, but due to false assumptions about payload correctness.

Specifically:

Provider payloads are hostile JSON

Newman output formats changed underneath us

Episodes are not arrays, but season-keyed maps

Keys are line-wrapped and fragmented

JSON parsers fail silently or catastrophically

Instead of attempting to “fix” JSON, the pipeline was re-architected to:

Treat payloads as text first,
detect signals instead of schema,
and extract independently and defensively.

This resulted in a cleaner, more resilient system than the original design.

2️⃣ What Failed (Root Causes)
❌ Assumption: “Payload is valid JSON”

Reality:

Keys split across lines ("epi\n sodes")

Mixed arrays and maps

Inconsistent nesting

Provider makes no schema guarantees

❌ Assumption: “Newman stdout is stable”

Reality:

Node emits warnings on STDERR

PowerShell treats STDERR as errors

JSON reporter output ≠ provider payload

❌ Assumption: “One extractor can parse everything”

Reality:

Episodes, seasons, info all express differently

Fan-out must be explicit, not inferred

3️⃣ Critical Pivot (The Moment Everything Changed)

The key insight:

Stop parsing JSON. Start scanning text.

Instead of fighting correctness, the pipeline now:

normalizes text

anchors on known signals

extracts opportunistically

fails loudly when anchors disappear

This approach absorbs unknown future shapes without breaking.

6️⃣ New Extraction Architecture (Authoritative)
🔑 Core Rule

Every extractor is independent, text-based, and single-purpose.

No extractor:

assumes valid JSON

depends on another extractor

mutates shared state

infers structure

7️⃣ Universal Normalization Step (MANDATORY)

Every extractor performs this step, after selecting the raw file:

$rawText = Get-Content $rawFile.FullName -Raw -Encoding UTF8
$scan    = ($rawText -replace '\s+', ' ')


Why:

rejoins broken keys

normalizes provider wrapping

stabilizes regex anchors

All matching is done against $scan.
$rawText is retained only for writing/debugging.

9️⃣ Why Extractors “Did Nothing” (But Were Correct)

When extractors logged:

No raw series file found


This was not failure.

Cause:

extractors looked in raw_store

router had moved files to pickup\default

Fix:

$RAW_INPUT_DIR = "C:\miratv_ingest\raw_store\pickup\default"


Once aligned, extraction resumed correctly.
