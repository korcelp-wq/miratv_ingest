1. Core Principle

Homegrown AI is not a model.
Homegrown AI is a loop.

The system becomes “alive” when a governed loop:

runs continuously or opportunistically

operates on real system artifacts

improves completeness and structure over time

respects authority boundaries

preserves uncertainty (perspective)

2. Perspective (Foundational Concept)

Perspective = the ability to hold uncertainty across time without collapsing it into certainty.

This is enforced through:

Provisional rule status

Confidence thresholds

Trend-based validation (usage over time)

Explicit canonization (never automatic)

Without provisional status, perspective cannot exist.

3. Governance Model (Finalized for Build Phase)
Rule States
discovered → provisional → canon → deprecated

Authority Rules

AI may propose rules, never promote or enforce them

Chats and parameter docs are legislative history, not law

Canon Boundary

Only rules explicitly present and active in igm_rules are authoritative.

4. Provisional Confidence Thresholds

Provisional rules accrue confidence through successful, non-friction usage.

Metrics

encounters

supports

overrides

confidence = supports / encounters

Guidance Thresholds

< 40% → likely eliminate

40–70% → remain provisional

> 70% → candidate for canon

> 90% sustained → constitutional-level stability

No automatic promotion. Ever.

5. The First Living Loop: Series Completion Loop (SCL)
Objective

Incrementally complete series metadata while:

6. Grinder-First Progressive Enrichment (GFPE) — Canon Strategy
Key Insight

If a file is still in the grinder, it is not a DB problem.

Grinder = local truth-in-progress

DB = authoritative curated state

AI operates locally on grinder artifacts

DB only receives shaped, final updates

7. GFPE Pipeline (Authoritative)
Phase 0 — Reality Check

Identify grinder failures, quarantined files, partial parses

No AI, no DB writes

Phase 1 — Grinder Salvage (Primary Path)

AI inspects raw local payloads

Explains grinder failure

Extracts data already present

Proposes grinder parsing fixes

No invention, extraction only

Phase 2 — Internal Source-of-Truth Enrichment

Use existing series patterns

Vector similarity for guidance only

Provisional inference allowed

Still no external calls

Phase 3 — Targeted External Lookup (Last Resort)

Surgical, field-specific

Only after local + internal exhaustion

Phase 4 — Acceptance

Partial knowledge is valid

Unknown remains unknown

No forced completeness

8. AI ↔ DB Boundary (Hard Rule)
AI (Local Only)

Reads:

grinder payloads

logs

schema introspection results

Produces:

grinder fix suggestions

structured DB update instructions

escalation reasons

DB (Remote / Server Only)

Accepts:

explicit INSERT / UPDATE

Enforces:

constraints

keys

provenance

Never sees:

raw failed payloads

half-parsed data

speculative structures
