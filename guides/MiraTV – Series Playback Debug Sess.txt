1. System Architecture Overview

MiraTV uses a three-tier architecture separating responsibilities.

Android App
     │
     ▼
Workers Gateway (miratv.club)
     │
     ▼
Xtream Provider Server
Layer Responsibilities
Layer	Role
Android App	UI, player, API calls
Workers Gateway	metadata, DB, caching
Provider (Xtream)	actual media streams
2. Servers Used
Backend / Activation Server

Used only for activation.

https://miratv.club/api.php
Gateway Server

Used for metadata and internal APIs.

https://miratv.club/_workers/ai/

Example endpoints:

series/concepts
series/concepts/by-category
series/concepts/episodes.php
Provider Server (Xtream)

Used ONLY for media streaming and Xtream APIs

http://uxurwymd.silvervpn.net:8080/

Important:
Port 8080 is required for streaming.

4. Xtream Streaming Paths

Only these root paths are valid.

/live
/movie
/series
/player_api.php
/get.php
/xmltv.php
/images
Live Channel Stream
/live/USERNAME/PASSWORD/STREAM_ID.ts

Example

http://uxurwymd.silvervpn.net:8080/live/Marina2025/3KY586YR/105.ts
Movie (VOD)
/movie/U/P/STREAM_ID.EXT

Example

/movie/Marina2025/3KY586YR/400671404.mkv
Series Episode

Correct format:

/series/U/P/EPISODE_ID

or

/series/U/P/EPISODE_ID.ext

Example:

http://uxurwymd.silvervpn.net:8080/series/Marina2025/3KY586YR/1327491

Important:

EPISODE_ID comes from get_series_info
5. MiraTV Android Playback Pipeline
Step 1 – Series Click
SeriesListActivity

Navigates to:

SeriesEpisodesActivity

Log example

Series clicked: 54332
Step 2 – Load Episodes
XtreamRepository.getEpisodes(seriesId)

Calls provider API:

player_api.php?action=get_series_info

Returns

episodes: Map<season, List<episode>>
Step 3 – Episode Click
SeriesEpisodesAdapter

Launches

PlayerActivity

With:

STREAM_URL
TITLE

Example log:

Playing episode: S01E01 (1327491)
Step 4 – Player Launch
PlayerActivity

Creates ExoPlayer instance.

Example log:

Initializing player with URL=http://.../series/U/P/1327491.mp4

Episodes load correctly but playback sometimes fails.

Player loads

7. Root Causes Identified
Issue 1 – Provider Port Missing

App normalized provider URL incorrectly.

Incorrect:

http://uxurwymd.silvervpn.net/

Correct:

http://uxurwymd.silvervpn.net:8080/

Fixed in:

SessionManager.getProviderUrl()
Issue 2 – Episode Extension Handling

Some Xtream servers require extension.

Examples:

.mkv
.mp4
.ts

Current strategy:

series/U/P/EPISODE_ID

Fallback:

series/U/P/EPISODE_ID.ext

Extension from:

providerEpisode.container_extension
Issue 3 – Some Provider Episodes Broken

Confirmed:

Episode 1 failed
Episode 2 worked

Meaning:

provider asset likely broken
