AI Integration & Document Ingestion — Architecture Realignment
Purpose of This Document

This document captures the evolution of thought, technical decisions, false starts, and final alignment reached while attempting to integrate AI processing into an existing ingestion-first system.

It serves as:

A record of why certain approaches failed

A justification for returning to file-based ingestion

A conceptual foundation for the Communication Matrix

A guardrail against repeating the same loop

Initial Goal

The original intent was to:

Introduce AI assistance into the system

Reduce workload via AI endpoints

Experiment with cheaper AI providers (e.g., Cohere)

Potentially route tasks dynamically between providers

Track execution via telemetry

This was initially approached through:

HTTP endpoints

JSON request/response

Direct streaming of text into AI calls

Newman / curl / bash-based invocation

Early Friction Signals
1. Transport-Level Instability

Repeated issues arose with:

curl behaving inconsistently

Newman not reliably executing requests

Content-Type handling inconsistencies

Token handling confusion

Silent failures (no DB writes, no durable artifact)

These issues were not logic bugs — they were transport and tooling fragility.

2. JSON as a False Convenience

JSON was initially used out of habit and external convention, but repeatedly caused problems:

Escaping issues

Parsing failures

Tool incompatibility

Debugging complexity

Zero benefit for text-only AI workloads

Conclusion reached explicitly:

JSON is not convenient.
It is actively harmful for this system.

Provider Drift & Adapter Confusion
Cohere Integration Issues

Several failures occurred around Cohere integration:

API response shape drift

Endpoint confusion (/classify vs /chat)

Account/model-specific payload differences

Mismatched function signatures

Duplicate parsing logic in orchestration layers

Key realization:

Provider quirks must live in provider adapters, not orchestration.

This led back to the existing (and correct) abstraction:

cohere_entry.php as the Cohere boundary

Provider adapters responsible for normalization

No Cohere logic leaking upward

Critical Realization: Everything in the DB Was Ingested as a File

At a certain point, a deeper pattern surfaced:

Every piece of data currently in the database arrived via file upload and ingestion.
None arrived via live streams or JSON APIs.

This was not accidental — it was a proven architectural invariant.

Attempts to bypass this invariant caused:

Invisible failures

Lack of retryability

No durable artifacts

Inability to reprocess

Debugging loops

This recognition broke the loop.

Why the Loop Happened
1. AI Enthusiasm

AI introduced a powerful illusion:

Fluent responses

Instant feedback

Apparent agreement

A sense that “this should just work”

This created shared enthusiasm, which masked architectural misalignment.

Key insight:

Enthusiasm is not the enemy.
Unexamined enthusiasm is the blind spot.

2. Context Drift

The system temporarily shifted frames:

From ingestion pipelines → API thinking

From documents → messages

From artifacts → streams

This drift conflicted with the system’s core strengths.

Reframing the Problem Correctly
Not a “Document Queue”

There was resistance to “reinventing document queuing,” correctly so.

Traditional queues:

Are ephemeral

Lose state

Hide failures

Optimize throughput over provenance

That is not what is needed.

What Is Actually Needed

The correct model is:

A document lifecycle ledger with workers

Components:

Filesystem as truth

Files exist or they don’t

States are visible

Reprocessing is possible

Database as history

Tracks what happened

Tracks attempts and outcomes

Never pretends something happened when it didn’t

Workers that pull

Claim work explicitly

Can be restarted safely

Do not depend on ephemeral delivery

The Communication Matrix (Clarified)

The Communication Matrix is not routing logic.

It is:

Metadata describing how a document communicates and how it should be interpreted.

Examples:

Communication Style	Examples	Processing Implication
Config / Parameters	YAML, INI	Deterministic parsing
Logs / Telemetry	System logs	Pattern extraction
Narrative	Design docs	Analytical reasoning
Creative	Fiction	Generative reasoning
Instructions	SOPs	Action-oriented processing

The matrix informs how workers read documents, not how messages are routed.

This concept only makes sense in a file-based ingestion model.

Why File-Based Ingestion Wins (Again)

File-based ingestion provides:

Determinism

Observability

Retryability

Reprocessability

Provenance

Calm debugging

Streaming input provides none of these reliably in this environment.

Conclusion:

AI does not get a special exception to ingestion rules.

Final Alignment
What Was Confirmed

The original ingestion-first architecture is correct

AI fits as a worker, not a destination

JSON is unnecessary and counterproductive

Files are the correct unit of truth

The Communication Matrix explains why this works

What Changed

No live AI endpoints as primary ingestion

No routing until execution is boring

No document queues

No streaming assumptions

What Improved

Architectural clarity

Reduced cognitive friction

Stronger trust in existing patterns

Earlier loop detection going forward

Guardrail for the Future

If progress exists only in language, endpoints, or configuration —
and not on disk or in tables — assume an illusion until proven otherwise.
