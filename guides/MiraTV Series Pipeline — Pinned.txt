📘 MiraTV Series Pipeline — Pinned Deterministic Model

Status: PINNED
Scope: Series ingest → normalize → exhaust deterministic work
Goal: Stable, extensible core with explicit AI insertion points
Design Rule: Filesystem = State Machine

1. Core Principle (Pinned)

The pipeline advances only when all deterministic work has been exhausted.

There is:

no timing logic

no progress inference

no semantic “completeness” checks

If work remains, a file exists.
If no work remains, no file exists.

2. Identity Invariant (Non-Negotiable)

series_id is immutable

Every file is scoped to exactly one series_id

No process invents or mutates identity

AI (future) must preserve identity

3. Responsibility Boundaries
Router

Purpose: Classification + emission
Responsibilities:

Inspect raw series payload

Detect which domains exist

Classify structure signature sufficient to select grinder

Emit .raw.json files

Exit immediately

Router does NOT:

wait

normalize

extract

decide progress

call next steps

Grinder

Purpose: Deterministic interpretation
Responsibilities:

Consume exactly one .raw.json input

Attempt deterministic extraction

Emit exactly one terminal artifact

Remove or archive the .raw.json

Exit

Grinder does NOT:

coordinate with other grinders

check global state

wait for other files

decide pipeline advancement

Deterministic Loop

Purpose: Exhaust deterministic obligations
Rule:

While any .raw.json files exist → work remains

When none exist → loop closes

The loop does not:

track timing

track order

track which file “finishes last”

4. Completion Model (Pinned)

The system does not track “completeness”.

It tracks closure of deterministic obligations.

A unit of work is closed when its .raw.json file is replaced by exactly one terminal artifact.

5. AI Integration (Deferred by Design)

AI is an upgrade path, not a dependency.

AI consumes unresolved artifacts (FAILED, QUARANTINE)

AI emits canonical replacements

AI re-opens work only by writing new .raw.json

Core loop remains unchanged

6. Why This Model Is Stable

No race conditions

No timing windows

No orphaned work

Safe restarts

Clear audit trail

PowerShell / Batch friendly

7. Pinned Statement (Keep This)

If a .raw.json exists, deterministic work remains.
If no .raw.json exists, deterministic work is exhausted.
