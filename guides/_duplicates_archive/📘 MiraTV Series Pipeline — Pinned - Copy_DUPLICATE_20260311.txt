📘 MiraTV Series Pipeline — Pinned Deterministic Model

Status: PINNED
Scope: Series ingest → normalize → exhaust deterministic work
Goal: Stable, extensible core with explicit AI insertion points
Design Rule: Filesystem = State Machine

1. Core Principle (Pinned)

The pipeline advances only when all deterministic work has been exhausted.

There is:

no timing logic

no progress inference

no semantic “completeness” checks

If work remains, a file exists.
If no work remains, no file exists.

2. Identity Invariant (Non-Negotiable)

series_id is immutable

Every file is scoped to exactly one series_id

No process invents or mutates identity

AI (future) must preserve identity
