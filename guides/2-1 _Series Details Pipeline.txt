🧨 Any filesystem logic inside Postman Tests breaks Newman

❌ Problem C — Payload extraction confusion

or Newman metadata only

5️⃣ Correct Series Info Flow (Final Design)
STEP 1 — DB selects next series IDs

PHP

SELECT id, provider_series_id
FROM series
WHERE details_ingested = 0
ORDER BY id
LIMIT 20;


This logic already exists in:

get_next_series.php

STEP 2 — Newman fetches ONE series at a time

Postman Request URL (NO logic in Tests):

{{base_url}}/player_api.php
  ?username={{username}}
  &password={{password}}
  &action=get_series_info
  &series_id={{series_id}}


✅ No Tests
✅ No file logic
✅ No mutation

STEP 3 — External wrapper captures payload

PowerShell wrapper (this is correct and REQUIRED):

newman run sCollection.postman_collection.json `
  --folder get_series_info `
  --env-var series_id=605 `
  --reporters cli `
  | Out-File "C:\MiraTV\export\series_605_raw.txt"


Then:

# Extract raw JSON body ONLY
(Get-Content series_605_raw.txt -Raw) |
  Select-String '{' -Context 0,10000 |
  Out-File series_605_payload.json -Encoding utf8


✔ This file must begin with {
✔ This file is what PHP expects

STEP 4 — PHP ingest consumes raw payload

Endpoint:

POST /_ingest/import_series_info_json.php


Headers:

X-Ingest-Token: <token>
Content-Type: application/json


Body:

series_605_payload.json

STEP 5 — DB flags completion
UPDATE series
SET
  details_ingested = 1,
  details_ingested_at = NOW(),
  is_dirty = 0
WHERE id = ?;
