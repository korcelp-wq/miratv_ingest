# 1. Executive Summary

The MiraTV application now successfully supports:

* VOD category browsing
* VOD listing
* VOD playback
* Series playback
* Xtream API compatibility layer

The backend architecture uses a **database-first content model**, where the application queries the internal database first and only resolves the provider stream during playback.

This architecture is intentional and correct because it:

* Allows local metadata enrichment
* Enables caching
* Supports AI-assisted indexing later
* Reduces provider API load
* Allows multi-provider aggregation

---

# 2. Final Working Architecture

```
Android App
     │
     │ HTTP
     ▼
player_api.php
(Xtream compatibility gateway)
     │
     │ PDO
     ▼
MySQL Stored Procedures
     │
     ▼
Content Tables
(vod, series, categories)
```

Provider systems are **not contacted until playback time**.

# 5. Critical Bug That Was Fixed

### Problem

Categories were returning the **local ID** instead of the **provider category ID**.

Example returned:

```
category_id = 2
```

But the VOD table stored:

```
category_id = 983
```

So the VOD query returned zero rows.

---

### Correct Fix

The categories stored procedure now returns:

```
provider_category_id AS category_id
```

# 6. Working Stored Procedures

## sp_xtream_get_vod_categories

```sql
DROP PROCEDURE IF EXISTS sp_xtream_get_vod_categories;
DELIMITER $$

CREATE PROCEDURE sp_xtream_get_vod_categories(
    IN p_username VARCHAR(255),
    IN p_password VARCHAR(255)
)
BEGIN
    SELECT COALESCE(
        JSON_ARRAYAGG(
            JSON_OBJECT(
                'category_id', CAST(provider_category_id AS CHAR),
                'category_name', name,
                'parent_id', COALESCE(parent_id, 0)
            )
        ),
        JSON_ARRAY()
    ) AS json_result
    FROM vod_categories
    ORDER BY name;
END$$

DELIMITER ;
```

---

## sp_xtream_get_vod_streams

```sql
DROP PROCEDURE IF EXISTS sp_xtream_get_vod_streams;
DELIMITER $$

CREATE PROCEDURE sp_xtream_get_vod_streams(
    IN p_username VARCHAR(255),
    IN p_password VARCHAR(255),
    IN p_category_id VARCHAR(50)
)
BEGIN
    SELECT COALESCE(
        JSON_ARRAYAGG(
            JSON_OBJECT(
                'num', CAST(v.vod_id AS CHAR),
                'name', v.title,
                'stream_type', 'movie',
                'stream_id', CAST(v.provider_vod_id AS CHAR),
                'stream_icon', COALESCE(v.poster_url, v.cover_url, ''),
                'rating', COALESCE(CAST(v.rating AS CHAR), ''),
                'rating_5based', '',
                'added', COALESCE(CAST(UNIX_TIMESTAMP(v.added_at) AS CHAR), ''),
                'category_id', CAST(v.category_id AS CHAR),
                'container_extension', 'mp4'
            )
        ),
        JSON_ARRAY()
    ) AS json_result
    FROM vod v
    WHERE p_category_id IS NULL
       OR p_category_id = ''
       OR CAST(v.category_id AS CHAR) = p_category_id;
END$$

DELIMITER ;
```

# 8. Important Architectural Rule

### Always Query Local Database First

Never call provider APIs directly for:

* category lists
* VOD lists
* series lists

Provider APIs should only be used for:

```
final playback stream resolution
```

This allows MiraTV to function as a **content intelligence platform** rather than a simple IPTV client.
