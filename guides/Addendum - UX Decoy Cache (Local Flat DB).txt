UX Decoy Cache (Local Flat DB)

Status: LOCKED
Priority: UX / Perceived Performance
Scope: Client-side only (Android / Local Runtime)

1️⃣ Purpose (Authoritative)

The Local Flat DB Cache exists solely as a UX distraction layer to prevent visible loading stalls during real data retrieval.

It is not a cache of record, not authoritative, and not part of the data pipeline.

2️⃣ Design Intent (Non-Negotiable)

Must render immediately

Must be small, static, and disposable

Must never block, delay, or influence real loading

Must be replaceable without ceremony

Must fail silently

If removed entirely, the system must still function correctly.

3️⃣ What It Is

✔ A minimal, read-only, local flat database (e.g., SQLite)
✔ Contains only browse scaffolding:

Categories

Home shelves

Thin shelf items (title + poster only)

✔ Used only to provide an instant first frame

4️⃣ What It Is NOT

❌ Not authoritative data
❌ Not synchronized with provider
❌ Not continuously updated
❌ Not subject to TTL enforcement
❌ Not locked, scheduled, or state-managed
❌ Not used for playback, entitlements, or decisions

Freshness is irrelevant.

5️⃣ Lifecycle Rules

App launches → Decoy renders immediately

Real loading begins asynchronously

As real data arrives:

UI replaces decoy content progressively

No blocking or dependency on decoy state

Decoy is silently abandoned

The decoy is never invalidated — it is simply overwritten by truth.

6️⃣ Failure Behavior (Required)
Failure	Outcome
Missing decoy DB	Ignore and continue
Corrupt decoy DB	Delete and continue
Schema mismatch	Ignore and continue
Slow load	Skip decoy entirely

Under no circumstances may decoy failure affect real loading.

7️⃣ Architectural Position
UI
 ├─ Decoy Layer (Local Flat DB) → Instant perception
 └─ Real Loading (API → DB → Provider) → Truth


The decoy layer is outside the control plane and outside job scheduling.

8️⃣ Governance Statement (Pinned)

The UX Decoy Cache exists to manage human perception, not system correctness.
It is intentionally dumb, disposable, and untrusted by design.

Any attempt to expand its responsibility violates this addendum.
