🧠 Key Realization (Root Cause)

EPG data was never JSON.

IPTV providers overwhelmingly serve XMLTV (XML) for EPG

Attempting to:

Convert XML → JSON

Chunk JSON

POST JSON payloads

…introduced unnecessary complexity and failure modes

Correct strategy:

Treat EPG as XML, stream-parse it server-side, and insert directly into MySQL.

🧱 Final Architecture (AUTHORITATIVE)
Provider EPG (XMLTV)
        ↓
PC (PowerShell / BAT)
        ↓  SCP / curl upload
Server /_ingest/epg.xml
        ↓
PHP XMLReader (streaming)
        ↓
MySQL (epg_programmes)


No provider calls from server.
No JSON conversion.
No memory-exhausting loads.

🔐 Security Model (Locked)

Ingest endpoints are token-protected

Token accepted via:

X-INGEST-TOKEN header OR

POST token= field (fallback)

Requests without a valid token return 403 Forbidden
