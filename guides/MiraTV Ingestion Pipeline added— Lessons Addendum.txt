1. Core Realization (Critical)
❌ Assumption That Broke Everything

“The EPG is JSON.”

It wasn’t.
It never was.

Most IPTV providers serve XMLTV (XML) for EPG — not JSON.

✅ Correct Mental Model

Providers serve XML

Servers do not magically convert formats

Any JSON step must be explicit

For large EPGs, JSON is the wrong format anyway

2. Correct Strategy (What Actually Works)
EPG Ingestion Strategy (Final)

Do NOT:

Convert XML → JSON

Chunk JSON in PowerShell

Push huge JSON payloads over curl

DO:

Upload the raw XMLTV file

Parse it server-side

Stream it using XMLReader

Insert rows incrementally into MySQL

This avoids:

Windows command-length limits

curl memory issues

JSON decode failures

timeouts on large payloads

3. Final EPG Flow (Authoritative)
Provider (XMLTV)
        ↓
Local machine (download epg.xml)
        ↓
BAT / PowerShell upload (curl -F)
        ↓
import_epg.php (server)
        ↓
XMLReader stream parse
        ↓
MySQL epg_programmes table

4. Authentication Reality (Important)
What Actually Works with curl + PHP

Headers were unreliable across environments.

Final working approach:

Send token as a POST form field

Not a custom HTTP header

curl -X POST ^
  -F "token=INGEST_TOKEN" ^
  -F "epg=@epg.xml" ^
  https://example.com/_ingest/import_epg.php

PHP Side
$INGEST_TOKEN = '...';
$token = $_POST['token'] ?? null;

if ($token !== $INGEST_TOKEN) {
    http_response_code(403);
    exit('Forbidden');
}

5. XML Parsing (The Only Scalable Way)
Why XMLReader

Constant memory

Handles 50k+ programmes

No need to load entire document

Designed for this exact problem

Working Pattern
$reader = new XMLReader();
$reader->open($xmlFile, null, LIBXML_PARSEHUGE);

while ($reader->read()) {
    if ($reader->nodeType === XMLReader::ELEMENT && $reader->name === 'programme') {

        $channel = $reader->getAttribute('channel');
        $start   = substr($reader->getAttribute('start'), 0, 14);
        $end     = substr($reader->getAttribute('stop'), 0, 14);

        $node = new SimpleXMLElement($reader->readOuterXML());

        $title = (string)($node->title ?? '');
        $desc  = (string)($node->desc ?? '');

    }
}

6. Database Truths (Hard Lessons)
Schema Must Match Reality

Common failures encountered:

channel_id vs channel

provider_channel_id not existing

Assumed foreign keys that weren’t there

Fix:
Insert what you actually have, not what you wish you had.

Example Working Columns
epg_programmes (
  provider,
  channel,
  start_time,
  end_time,
  title,
  description
)
